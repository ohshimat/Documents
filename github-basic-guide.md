# GitHub基本操作ガイド

## 1. リポジトリの作成・初回コミット＆プッシュ
- GitHub上で新規リポジトリを作成する。
- ローカルで作成したファイル／フォルダを初期コミットし、`git push`でGitHubにアップロード。

## 2. リポジトリのクローン（新しいフォルダにチェックアウト）
- GitHubリポジトリページで「Code」ボタン → URLをコピー。
- クローン先フォルダで以下を実行：
  ```bash
  git clone <コピーしたURL>
  ```
- クローン後、希望のブランチに切り替える場合は：
  ```bash
  git checkout <ブランチ名>
  ```

## 3. 「リポジトリ」と「プロジェクト」の違い
| 用語         | 主な役割         | 保管内容                | 管理対象                  |
|--------------|------------------|-------------------------|---------------------------|
| リポジトリ   | ファイル管理     | コード・データ・履歴    | 1つの成果物/コード単位    |
| プロジェクト | タスク/進行管理 | Issue / Pull Request    | 複数リポジトリ/全体作業   |

- リポジトリ：コードと履歴管理の単位
- プロジェクト：タスク&進捗管理のボード

## 4. 「Pull」はどういうときに使う？
- GitHubリポジトリの**最新の変更を取得したいとき**に使う。
- チームで共同開発時、他の人の更新を反映させたいとき。
- プッシュや作業前、**競合（conflict）を防ぐ**ためにも活用。

```bash
git pull
```
- 上記でリモートの最新状態をローカルに取り込む。

## 5. 補足コマンドとtips
- `git fetch`：リモートの最新情報のみ取得（マージはしない）。
- `git merge`：取得済みの変更をローカルのブランチに反映。
